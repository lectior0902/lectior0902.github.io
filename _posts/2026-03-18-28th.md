# 21강. 과적합 방지 (Overfitting Prevention)

딥러닝 모델의 성능을 결정짓는 핵심 요소 중 하나인 **과적합(Overfitting)**의 정의와 이를 해결하기 위한 다양한 기법들 정리

---

## 1. 과적합(Overfitting)이란?
모델이 **학습 데이터에 너무 과하게 적합**되어, 실제 새로운 데이터(시험 데이터)가 들어왔을 때 예측 성능이 현저히 떨어지는 현상을 의미함.

* **과소적합(Underfitting):** 모델이 너무 단순하여 학습 데이터조차 제대로 학습하지 못한 상태
* **적정(Balanced):** 학습 데이터와 시험 데이터 모두에서 낮은 오차를 보이는 상태
* **과적합(Overfitting):** 학습 데이터에 대한 오차는 매우 낮지만, 시험 데이터에 대한 오차는 큰 상태

---

## 2. 과적합 해결 방법

### 2.1 데이터 측면의 해결책

* **데이터 확보:** 데이터의 양이 많아질수록 모델은 특정 샘플의 노이즈가 아닌 일반적인 패턴을 학습하게 됨.
* **데이터 증식 (Data Augmentation):** 기존 데이터를 변형하여 데이터셋의 규모를 키우는 기법.
    * 예: 이미지 평행이동, 대칭이동(Flip), 회전, 밝기 조절 등



### 2.2 학습 제어 측면
* **조기 종료 (Early Stopping):** 학습 반복 횟수(Epoch)가 늘어남에 따라 학습 데이터의 손실은 계속 줄어들지만, **시험 데이터의 손실이 증가하기 시작하는 지점**에서 학습을 강제로 중단하는 기법.



### 2.3 모델 구조 및 가중치 조절 (Regularization)
모델의 복잡도를 줄여 일반화 성능을 높이는 방법.

* **L1 / L2 정규화 (Regularization):** 손실 함수에 가중치의 크기를 제어하는 항을 추가함.
    * **L1 정규화:** 가중치의 절대값 합을 추가 (Sparse 모델 생성)
    * **L2 정규화:** 가중치의 제곱 합을 추가 (가중치를 작게 유지, Weight Decay)
* **드롭아웃 (Dropout):** 학습 시 신경망의 일부 노드를 무작위로 비활성화하여 특정 노드에 편향되는 것을 방지함.
* **드롭커넥트 (DropConnect):** 노드가 아닌 노드 사이의 **연결 가중치(Weight)**를 무작위로 0으로 만듦.



### 2.4 고급 최적화 기법
* **교란라벨 (DisturbLabel):** 학습 데이터 중 일부의 라벨을 의도적으로 틀리게 제공하여 모델이 학습 데이터에 너무 집착하지 않게 함.
* **배치 정규화 (Batch Normalization):** 각 층의 활성화 값을 정규화하여 학습 과정을 안정화하고 과적합을 억제함.
* **라벨 스무딩 (Label Smoothing):** 정답 라벨을 1(100%)로 두지 않고 작은 값($\alpha$)만큼 나누어 주어 모델의 과도한 확신을 완화함.
    * 예: [0, 1, 0] → [0.033, 0.933, 0.033]

# 22강. 데이터 불균형 (Imbalanced Data)

데이터 세트 내 클래스의 분포가 비대칭적인 상황에서 발생하는 문제점과 이를 해결하기 위한 주요 기법들 정리.

---

## 1. 데이터 불균형의 정의
* **정의**: 데이터 세트 내의 클래스 분포가 균일하지 않고 특정 클래스가 압도적으로 많은 상태.
* **불균형 데이터의 예**:
    * 정상 거래 vs 사기 거래 (FDS)
    * 기체 결함 탐지
    * 높은 클릭률(CTR) vs 정상 클릭률
    * 희귀 질병 vs 일반 질병



---

## 2. 데이터 불균형 해결 방법

### 2.1 데이터 증식 (Data Augmentation)
* 소수 클래스의 데이터를 변형(회전, 대칭 등)하여 데이터의 양을 늘림.

### 2.2 샘플링 기법 (Sampling)
* **언더 샘플링 (Under-sampling)**: 다수 클래스의 데이터를 소수 클래스 수준으로 줄임. 데이터 손실의 위험이 있음.
* **오버 샘플링 (Over-sampling)**: 소수 클래스의 데이터를 복제하여 늘림. 과적합(Overfitting)의 위험이 있음.
* **SMOTE (Synthetic Minority Over-sampling Technique)**: 소수 클래스 데이터 사이의 선분 위에서 가상의 데이터를 생성하여 과적합을 방지하며 증식함.



### 2.3 가중 손실 함수 (Weighted Loss Function)
* 손실 함수 계산 시 각 클래스에 부여하는 가중치를 조절함.
* 데이터 수가 적은 소수 클래스에 높은 가중치를 부여하여 모델이 해당 클래스를 더 중요하게 학습하도록 함.

### 2.4 평가 기준 정의 (Evaluation Metrics)
* 데이터가 불균형할 경우 일반적인 '정확도(Accuracy)'는 모델의 성능을 왜곡할 수 있음.
* **대안 지표**: 
    * 정밀도 (Precision)
    * 재현율 (Recall)
    * F1-Score
    * ROC 커브 및 AUC

# 23강. 성능 향상을 위한 튜닝 (Model Tuning)

딥러닝 모델의 예측 성능을 극대화하기 위해 조정할 수 있는 주요 하이퍼파라미터 및 설정 요소들 정리.

---

## 1. 모델 구조 및 설계 관련
* **레이어 및 필터 설정**: 레이어 수, 필터 수, 풀링(Pooling) 방식, 필터 크기 등을 조정하여 모델의 용량과 복잡도를 결정함.
* **이미지 크기**: 입력 이미지의 해상도를 조절하여 특징 추출의 정밀도를 변경함.
* **활성화 함수 (Activation Function)**: ReLU, Sigmoid, Tanh 등 적절한 함수를 선택하여 비선형성 및 기울기 소실 문제를 제어함.

---

## 2. 학습 프로세스 관련
* **에폭(Epoch) 및 배치(Batch) 수**: 전체 데이터 학습 반복 횟수와 한 번에 학습할 데이터 묶음의 크기를 설정함.
* **최적화 알고리즘 (Optimizer)**: Adam, SGD, RMSprop 등 모델의 가중치를 업데이트할 알고리즘을 선택함.
* **학습률 (Learning Rate)**: 가중치 업데이트의 보폭을 결정함. 너무 크면 발산하고, 너무 작으면 학습 속도가 매우 느려짐.

---

## 3. 최적화 및 정규화 관련
* **과적합 방지**: 모델이 과적합된 경우 드롭아웃(Dropout), L2 정규화 등을 적용함.
* **데이터 증식 (Data Augmentation)**: 학습 데이터가 부족하거나 다양성이 필요할 때 데이터를 변형하여 추가함.
* **샘플링 방법**: 언더샘플링(Undersampling)이나 가중 샘플링(Weighted Sampling) 등을 통해 데이터 불균형 문제를 해결함.

---

## 4. 손실 함수 및 초기화
* **손실 함수 (Loss Function)**: 문제 유형(분류, 회귀 등)에 맞는 목적 함수를 설정함.
* **초기 모델 파라미터 (Initial Parameters)**: 가중치 초기화 방법(Xavier, He 등)에 따라 학습의 수렴 속도와 최종 성능이 달라질 수 있음.

---

# 24강. 전이 학습 (Transfer Learning)

기존의 학습된 지식을 새로운 문제 해결에 활용하여 효율성을 극대화하는 전이 학습의 개념과 주요 종류를 정리함.

---

## 1. 전이 학습의 정의
* **개념**: 기존에 알고 있는 다른 지식(Source Domain)을 활용하여 새로운 문제(Target Domain)를 해결하는 학습 방법임.
* **장점**: 
    * 보다 적은 데이터 양으로도 높은 성능 개선이 가능함.
    * 처음부터 학습하는 것에 비해 학습 시간을 획기적으로 절약할 수 있음.



---

## 2. 전이 학습의 주요 종류

### 2.1 인스턴스 기반 (Instances-based)
* 소스 도메인의 데이터 중 타겟 도메인에 도움이 될만한 데이터에 적절한 가중치를 부여하여 직접 활용하는 방식임.

### 2.2 매핑 기반 (Mapping-based)
* 소스 도메인과 타겟 도메인의 유사성을 이용하여 두 도메인을 공통된 공간으로 매핑하고, 새로운 데이터를 생성하여 학습에 활용함.

### 2.3 네트워크 기반 (Network-based)
* **가장 많이 사용되는 방식**으로, 소스 도메인에서 학습된 네트워크의 구조와 가중치(Pre-trained Model) 일부를 가져와 타겟 도메인에 맞게 재구성함.
* 주로 앞단(Feature Extraction)의 층은 고정(Freeze)하고 뒷단의 층을 새로 학습함.



### 2.4 적대적 기반 (Adversarial-based)
* 적대적 학습 기법(GAN 등)을 활용하여 소스 도메인과 타겟 도메인 모두에서 공통적으로 적용 가능한 특징을 찾아내어 전이하는 방식임.
