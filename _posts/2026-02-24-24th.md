# 📌 NoSQL이란 무엇인가?

NoSQL(Not Only SQL)은 전통적인 관계형 데이터베이스(RDBMS)와 달리  
**스키마가 고정되지 않고, 비정형·반정형 데이터를 유연하게 저장**할 수 있는 데이터베이스이다.

대표적인 NoSQL 종류에는 다음과 같은 유형이 있다.

- Document DB (예: MongoDB)
- Key-Value Store (예: Redis)
- Column Store (예: Cassandra)
- Graph DB (예: Neo4j)

대규모 트래픽, 빅데이터, 실시간 서비스 환경에서 자주 활용된다.

---

# ✅ NoSQL의 장점

### 1️⃣ 유연한 스키마 (Schema-less)

컬럼 구조를 미리 엄격하게 정의하지 않아도 되어 데이터 구조 변경이 빈번한 서비스에 유리하다.

### 2️⃣ 수평적 확장(Scale-Out)

서버를 추가하는 방식으로 확장이 가능해 대규모 트래픽 처리에 적합하다.

### 3️⃣ 빠른 읽기/쓰기 성능

특정 목적에 맞춰 설계되어 캐싱, 실시간 처리에 강점이 있다.

---

# ❌ NoSQL의 단점

### 1️⃣ JOIN 기능의 한계

관계형 DB처럼 복잡한 JOIN이 어렵거나 비효율적일 수 있다.

### 2️⃣ 트랜잭션 지원 제약

ACID 특성이 제한적이거나 구현 방식이 다를 수 있다.

### 3️⃣ 데이터 중복 가능성

정규화 구조가 약해 데이터 중복이 발생하기 쉽다.

---

# 📊 RDBMS vs NoSQL 비교

| 구분 | RDBMS | NoSQL |
|------|--------|--------|
| 스키마 | 고정 스키마 | 유연한 스키마 |
| 확장 방식 | 수직 확장 (Scale-Up) | 수평 확장 (Scale-Out) |
| JOIN | 강력 | 제한적 |
| 트랜잭션 | 강력한 ACID 지원 | 일부 제한 |
| 사용 사례 | 금융, ERP | SNS, 빅데이터, 실시간 앱 |

---

# 🎯 언제 NoSQL을 선택해야 할까?

- 데이터 구조가 자주 변경될 때
- 대규모 트래픽이 예상될 때
- 빠른 개발과 확장이 필요할 때
- 비정형 데이터(로그, JSON 등)를 많이 다룰 때

반대로, **정확한 트랜잭션 관리와 복잡한 관계 모델링이 중요하다면 RDBMS가 더 적합**하다.

---

# 📌 SQL과 NoSQL이란?

데이터베이스는 크게 **SQL(RDBMS)** 과 **NoSQL** 로 구분된다.

- **SQL**: 관계형 데이터베이스(Relational Database)
- **NoSQL**: 비관계형 데이터베이스(Not Only SQL)

두 기술은 **사용 목적과 서비스 환경에 따라 선택이 달라지는 기술**이다.

---

# 🧱 SQL (RDBMS)의 특징

- 테이블 기반의 구조
- 고정된 스키마
- 강력한 ACID 트랜잭션 지원
- JOIN을 통한 복잡한 관계 처리 가능

대표 예시:
- MySQL
- PostgreSQL
- Oracle
- SQL Server

금융, ERP, 정산 시스템처럼 **정확성과 일관성이 중요한 서비스**에 적합하다.

---

# 🌐 NoSQL의 특징

- 스키마 유연성
- 수평 확장(Scale-Out)에 강점
- 비정형 데이터 처리에 유리
- JOIN 기능은 제한적

대표 유형:
- Document DB (MongoDB)
- Key-Value Store (Redis)
- Column DB (Cassandra)
- Graph DB (Neo4j)

SNS, 로그 분석, 대규모 트래픽 서비스에 적합하다.

---

# 📊 SQL vs NoSQL 비교표

| 구분 | SQL (RDBMS) | NoSQL |
|------|-------------|--------|
| 데이터 구조 | 테이블(행·열) | 문서, Key-Value, 그래프 등 |
| 스키마 | 사전 정의, 고정 | 유연, 변경 용이 |
| 확장 방식 | 수직 확장(Scale-Up) | 수평 확장(Scale-Out) |
| 트랜잭션 | 강력한 ACID 지원 | 일부 제한적 |
| JOIN | 매우 강력 | 제한적 또는 비효율 |
| 데이터 일관성 | 강함 | 경우에 따라 최종 일관성(Eventual Consistency) |
| 사용 사례 | 금융, 정산, ERP | SNS, 빅데이터, 실시간 서비스 |

---

# 🎯 언제 무엇을 선택해야 할까?

### ✅ SQL이 적합한 경우
- 복잡한 관계형 구조
- 데이터 무결성 중요
- 트랜잭션 관리 필수
- 정형 데이터 중심

### ✅ NoSQL이 적합한 경우
- 데이터 구조가 자주 변경
- 대규모 트래픽 처리 필요
- 빠른 확장 및 유연성 중요
- 비정형 데이터 다수 존재
